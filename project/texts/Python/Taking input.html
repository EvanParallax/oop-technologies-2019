Sometimes programs need to interact with users, either to get data or to provide some sort of result. And that's when the input function comes ahead.

Reading input from a user

The input data we want to get is nothing but some value from the user. The input function reads this value and returns it in a program as a string. As an example, the following program reads the user name and prints a greeting.

user_name = input()
print('Hello, ' + user_name)

In the first line, the program will wait for the user to enter something and the fact that we assign the input to the variable will let us perform some actions with it later. In the second line, the program appends the entered name to the end of 'Hello, ' string and print the whole result.

If a user enters Sauron, this program prints:

Hello, Sauron

So, your program printed a result that depends on the entered data (name).

Clear messages

It is highly recommended to show clear messages if we want the user to understand what (s)he is supposed to do while the program waits for an input. For this purpose, the input function may take an optional argument representing a message:

user_name = input('Please, enter your name: ')
print('Hello, ' + user_name)

After starting this program, the user will see the message, enther his/her name and get the result:

Please, enter your name: Sauron
Hello, Sauron

Alternatively, we may print the message for the user separately: 

print('Enter your name: ')
user_name = input()
print('Hello, ' + user_name)

The difference isn't that significant: in the previous example, the input will be printed in the same line as the message, while in this case it will be written in the next line. So, you may choose whatever you like.

[ALERT-warning]Although it is recommended to print messages for users, avoid them in our educational programming challenges, otherwise your code may not pass our tests.[/ALERT]

Important details

But how long can the user's input be? And how will the program understand that the person has written everything (s)he wanted?

That's actually an important point about the input() function: as soon as the program has run into this function, it stops and waits for the user to write something and press Enter. That also means that if there's no input from the user, it will not execute any further.

Another important point is that anything you enter will be regarded by the function as a string. Whether you enter digits or letters, the input will be converted to a string.

If you want a number to be a number, you should write it explicitly:

print("What's your favourite number?")
value = int(input())  # now value keeps an integer number

However, be careful: in these circumstances, if a user writes not an integer, an Error will appear.
