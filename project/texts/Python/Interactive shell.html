A good programmer doesn’t have to know everything, but what he has to know is where to find and how to test the needed information. Nowadays, in the era of the Internet the answer to “where” is quite straightforward - on the Internet, of course! In this article, we’ll partly talk about the Internet, but mostly - about the “how” part.

When you start programming, there are plenty of cases when you need to test something before implementing it in your code. It would be rather inconvenient to create a new scratch file every time you need to check a type of a variable or what a function returns, what kind of data you are working with etc. To avoid creating unnecessary files, Python has a great tool for executing small (or large) pieces of code. This tool’s known under different names: the Python interactive console, the Python interpreter or Python shell. 

Make sure you have installed Python

Before accessing the shell, you have to make sure that you have already installed Python. If not, please, follow the detailed instructions from here. To make sure everything's correct, you can execute the following command in a terminal / command line:

python --version

The output should be something like this, but with your version of Python:

Python 3.6.7

Interactive interpreter capabilities

When you enter the shell, the first thing you see is a welcome message, which specifies the version of Python, information about your operating system along with some commands you can enter to get additional information.
The Python interactive interpreter accepts Python syntax, which you place after:

  • the primary prompt, which starts with three greater-than signs (>>>);

  • or the secondary prompt, which start with three dots (...).

Every new command to execute starts with the primary prompt. For example, we can assign values to variables:

>>> a = "call me"

Once you have written and executed your code by hitting the “Return” button, you will see the result of your command and another primary prompt:

>>> a = "call me"
>>> 

You can use the shell for making calculations:

>>> 5 + 12
17
>>> 17 * 2
34

It's quite handy if you want to test something, say list operations:

>>> a = [1,2,3]
>>> b = [1,2]
>>> a + b
[1, 2, 3, 1, 2]
>>> a.append(b)
>>> a
[1, 2, 3, [1, 2]]

It's quite likely that for your experiments you will need something more than just Python built-ins. You are free to use import statements in the interactive mode:

>>> import random
>>> random.randint(0, 100)
65
>>>

Sometimes you can check the documentation of an object or the methods available for a specific object without even going to the Internet by just executing the following commands in the Interactive shell:

help(list)

dir(tuple)

Multi-line constructions

The secondary prompt is used in multi-line constructions, like:

>>> construction_is_multiline = True
>>> if construction_is_multiline:
...     print("You will see the secondary prompt after executing the first line”)

You don’t have to print everything in the interactive mode. In fact, if your line of code is a data structure, executing it will print this data into the command line as it is:

>>> mother_called = 10
>>> if mother_called > 5:
...     "go home immediately!"
... else:
...     "you will have problems"

Another thing is that when it comes to multi-line programs, you have to make indentations according to Python syntax. The only allowance here is you don’t have to make indentations a multiple of four spaces, so the following code will be executed successfully too:

>>> mother_called = 10
>>> if mother_called > 5:
...               print("go home immediately!")
... else:
...      print("you will have problems")

The main thing is to keep the indentation of instructions on the same level. Of course, you should always aim to follow the style, so it's always better to write clean code with correct indentations. 

The following code will throw a syntax error because else is not on the same level with if, to which it belongs:

>>> mother_called = 10
>>> if mother_called > 5:
...               print("go home immediately!")
...   else:
...      print("you will have problems")

Navigation in Python shell

After you're done with experimenting, you might need to continue working in command line or terminal. To quit Python shell, press CTRL + Z or CTRL + D. 

CTRL + Z will leave a trace in your command line:

>>> 
[5]+  Stopped                 python
user:~$

 while CTRL + D is a silent exit, which just stops the interactive mode:

>>> 
user@compuhter-back:~$

Another way of quitting the interactive mode is to execute quit() or exit() command.

CTRL +C is a KeyboardInterrupt, which allows you breaking the multiline structures or infinite loops.  

Since Python 3.4, the interactive mode session is being logged into a .python_history file on *nix OS, so quit() function leaves a trace in this file, while hotkeys don't.

If you need to clean up the working space in your command line, you can use CTRL + L, which creates the necessary for your screen size amount of empty lines.
