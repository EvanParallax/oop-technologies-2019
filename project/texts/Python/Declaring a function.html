Definition

Often enough, built-in functions cannot suffice even beginners. In such a case, there is no choice but to create your own function using the keyword def (right, derived from define). Let's take a brief look at the syntax:

def function_name(argument0, argument1):
    # function's body
    return argument0 * argument1


a = function_name(3, 5)  # 15
b = function_name(a, 10)  # 150

After def we write the name of our function (so as to invoke it later) and the arguments enclosed in parentheses. Do not miss the colon at the end. Names of a function and its arguments follow the same convention as variable names, that is, they should be written in lowercase with underscores between words. The argument names are assigned to the values passed in parentheses at the function call. An indent of 4 spaces shows the interpreter where the function's body starts and where it ends. Only when the indentation is absent, the definition of the function ends.

Execution and return

The keyword return is used to indicate what values the function outputs (the product of the arguments in our previous example). Basically, it is the result of the function call. So, a refers to the value 15 and b to 150. Importantly, Python interpreter stops performing the function after return. But what if the function body contains more than one return statement? Then the execution will end after the first one.

One more thing to say is that functions do not have to return values. The well-known print() function does not, in fact, return anything. Examine the code below:

chant = print("We Will Rock You")
print(chant)
type(chant)

And its output:

We Will Rock You
None
<class 'NoneType'>

We declared the variable chant and invoked print(). Obviously, the function was executed. But the variable itself turned out to be the None object, which means the called function had nothing to return. This value is of Nonetype.

Arguments

Each time we call a function, we initialize its arguments again (so that all the calculations start from scratch). There are different ways to assign the arguments:

  • Positional arguments (alias non-keyword) are passed to a function by position, from left to right. 
    def count_args(x, y):
    print("The first argument is", x)
    print("The second one is", y)


count_args(0, 1)
# The first argument is 0
# The second one is 1
count_args(1, 0)
# The first argument is 1
# The second one is 0

    The order of the arguments determines how they are assigned.

  • Keyword arguments are matched by name. 
    def print_args(x, y):
    print(x, y)


print_args(x=1, y=2)  # 1 2
print_args(y=3, x=2)  # 2 3

    The order doesn't matter here, but keyword arguments are always written after non-keyword arguments:

    print_args(1, y=3)  # 1 3
print_args(x=1, 3)  # SyntaxError: positional argument follows keyword argument
  • Defaults are specified in a function definition and represent default values for arguments that are not passed. 
    def location(place, planet="Earth"):
    print(place, "on", planet)


location("Berlin")  # Berlin on Earth
location("Breakfast", planet="Pluto")  # Breakfast on Pluto
location("Craters", "Mercury")  # Craters on Mercury

    Arguments with default values are optional in some way. However, new values can be assigned to them either by name or by position.

Finally, the syntax in Python enables us to pass all items from a sequence as individual positional arguments using *. We can also get key-value pairs from a dictionary and pass them as keyword arguments using **:

def say_bye(x, y):
    print("Bye,", x)
    print("Au revoir,", y)


names = ["Peter", "Sophie"]
say_bye(*names)
# Bye, Peter
# Au revoir, Sophie
dict_names = {'y' : "Laura", 'x' : "Robin"}
say_bye(**dict_names)
# Bye, Robin
# Au revoir, Laura
