Python provides special notes to explain what and why is happening in certain lines of code. Such notes are called comments. They are especially useful for beginners. Throughout this course, we will often use comments to explain our examples.

What is a comment?

Any string starting with a hash (#) is considered as a comment by Python, and everything after the hash mark and up to the end of the line will be ignored when running the code.

print("This will run.")  # This won't run

The one in the example above is called inline comment because it follows a piece of code in the same line. Another kind is a block comment that refers to the code that follows it:

# printing three numbers
print("1")
print("2")
print("3")

Now we intend to use the comments primarily for learning purposes. Pros and cons of using comments in real programs will be discussed in other topics.

Formatting the comments

Although it is pretty easy to write a comment, let's discuss how to do this in accordance with best practices.

To begin with, after a hash mark there should be one space, and in inline comments, there should be two spaces between the end of the code and the hash mark.

print("Learning Python is fun!")  # This is a proper comments' formatting
print("PEP-8 is important!")#This is a very bad example

A block comment is indented at the same level as a statement to mark. E.g. the following example is wrong:

    # this comment is at a wrong place
print("This is a statement to print.")

It is a good practice to split a long comment into several lines: you can do so either by adding a hash mark at the beginning of each new line:

# Imagine that this is an example of a really long comment
# that we need to spread over three lines, so we continue
# to write it even here
print("The long comment above explains this line of code.")

We hope our comments will help you understand our code examples!
