General considerations

When moving from simple single-line programs to more complicated ones there is another question appearing: how to write code that is clean and easy to read. It may seem not very important at the beginning, but because in real life programming is a process that involves a lot of people working together, you spend more time reading code than writing it.

Although Python is often more readable than other programming languages because of its minimalistic syntax, just syntax itself is not enough for programs to be well readable. The way you write code affects general readability a lot. So it makes sense to follow common conventions about code style in order for your code to be understood by other programmers.

But where do these conventions come from? In Python, there is such document as PEP 8. The key idea of this document is to use the same code style for all python projects, as if they were written by the same programmer. It makes easy to understand such projects by any developers, especially beginners. 

PEPs

Before going further, let’s say a bit about PEPs. PEP stands for Python Enhancement Proposal. There are different types of PEP and the most useful one for beginners is informational PEP. PEPs of this kind typically describe commonly accepted guidelines or conventions about the language and may be very helpful. Besides PEP 8, that is an official style guide, another great PEP to check out is the Zen of Python.

Now, when we know what PEP 8 is, let’s cover some recommendations it provides.

The length of a line

Do not use more than 79 characters in a line of code. Short lines are easier to observe in code editors. Throughout this course we will learn different ways to achieve it.

Avoid extra spaces

Sometimes you may add some spaces where they are not actually needed. This will reduce the readability of your code.

  • Avoid extra spaces within parentheses.

Good:

print('Hello!')

Bad:

print( 'Hello!' )
  • Avoid an extra space before an open bracket.

Good:

print('some text')

Bad:

print ('some text')

Quotes

We already mentioned earlier that you can use either single or double quotes to define strings. To be consistent choose one that you like and use them all along in your code. PEP 8 only states that if string contains single or double quotes, you should use the other one to avoid backslashes.

Good:

print("It's a good string!")

Bad and harder to read:

print('It\'s a bad string!')

You can notice that the first example is more readable.

What’s next?

Further, you will learn a lot of concepts about Python and become more powerful as a programmer, but following the code style will always remain important. We strongly recommend you to always check PEP 8 after learning something new. It will help you to build a skill of writing clean and readable code.

Do not try to learn all the conventions at once; just open them from time to time after learning some new concepts. We will also give parts of these conventions in this course.
