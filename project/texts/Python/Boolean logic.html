Boolean type

Boolean type or simply bool is a special data type that has only two possible values: True and False. In Python bool value names start with a capital letter.

Boolean or logical type is a common way in programming languages to represent something that has only two opposite states like on or off, yes or no, etc.

If you are writing an application that keeps track of door's openings you'll find it natural to use bool to store current door state.

is_open = True
is_closed = False

print(is_open)  # True
print(is_closed)  # False

Boolean operations

There are three built-in boolean operators in Python: and, or and not. The first two are binary operators which means that they expect two arguments. not is an unary operator, it is always applied to a single operand.

  • and is a binary operator, it takes two arguments and returns True if both arguments are True, otherwise, it returns False. 
    a = True and True  # True
b = True and False  # False
c = False and False  # False
d = False and True  # False
  • or is a binary operator, it returns True if at least one argument is True, otherwise, it returns False. 
    a = True or True  # True
b = True or False  # True
c = False or False  # False
d = False or True  # True
  • not is an unary operator, it reverses the boolean value of its argument. 
    to_be = True  # to_be is True
not_to_be = not to_be  #  not_to_be is False

The precedence of boolean operation

Logical operators have a different priority and it affects the order of evaluation. Here are the operators in order of ascending of their priorities: or, and, not. Having this in mind, consider the following expression:

a = False or not False  # True, because not False evaluates first

Although, while dealing with bare boolean values things may seem obvious, the precedence of logical operations will be very important to remember when you start working with so-called truthy and falsy values.

Truthy and falsy values

Though Python have native boolean data type we often want to use non-boolean values in a logical context. And Python lets you test almost any object for truthness. Values of non-boolean types (integers, strings, etc.) when used with logical operations are called either truthy or falsy depending on whether they are considered True or False. Here is what generally considered falsy in Python:

  • constants defined to be false: None and False.

  • zero of any numeric type: 0, 0.0.

  • empty sequences and collections: '', [], {}.

Anything else is considered truthy.

Now we can more clearly demonstrate the difference in operators priority:

truthy_integer = False or 5 and 100  # 100, because and has a higher priority than or

Generally speaking, and and or could take as arguments and return anything that can be tested for truth value. Remember, or and and return one of their operands, not necessarily of a Boolean type. But not always returns bool.

Another tricky example involving an empty collection:

tricky = not (False or '')  # True

Let's break it into parts. A pair of parentheses is a way to create a tuple, so '' represents an empty tuple that is considered falsy just like any other empty collection. Since the left argument of or is also falsy it returns ''. Then the result of the enclosed expression is negated and finally we get True, because not operator always returns Boolean values.

Short-circuit evaluation

The last thing to mention is that logical operators in Python are short-circuited or lazy. That means that the second operand is only evaluated if the first one is not sufficient to evaluate the whole expression.

  • x and y returns x if x is false, otherwise, it evaluates and returns y

  • x or y returns x if x is true, otherwise, it evaluates and returns y

For instance:

lazy_or = True or (1 / 0)  # division is never evaluated, because the first argument is True
lazy_and = False and (1 / 0)  # division is never evaluated, because the first argument is False
