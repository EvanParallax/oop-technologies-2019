Inside a program, you can write special text which will be ignored by the compiler known as comments. They allow you to exclude code from the compilation process (disable it) or clarify a piece of code to yourself or other developers.  

Kotlin supports three kinds of comments.

End-of-line comments

The compiler ignores any text from // to the end of the line.

fun main(args: Array<String>) {
    // The line below will be ignored
    // println("Hello, World")

    // This prints the string "Hello, Kotlin
    println("Hello, Kotlin") // Here can be any comment
}

In the example above the text after // is ignored by the compiler.

Multi-line comments

The compiler ignores any text from /* to the nearest */. It can be used for single and multiple-line comments.

fun main(args: Array<String>) {
    /* This is a single-line comment */
    /*  This is an example of
        a multi-line comment */

    /* All lines below will be ignored
        println("Hello")
        println("World")
    */
}

It is possible to include comments in other comments.

fun main(args: Array<String>) {
    /*
    System.out.println("Hello") // print "Hello" 
    System.out.println("Kotlin")  // print "Kotlin"
    */
}

The part of the code above is ignored by the compiler because of the comments enclosed in  /* ... */.

Documentation comments (doc comments)

The compiler ignores any text from /** to */ just like it ignores multi-line comments.

This kind of comments can be used to automatically generate documentation about your source code using a special tool. Usually, these comments are placed above declarations of some program elements. Also, in this case, some special labels such as @param, @return and others are used for controlling the tool.

See the example below.

/**
 * The main method accepts string arguments from outside
 *
 * @param args from the command line
 */
fun main(args: Array<String>) {
    // do nothing
}

Do not be afraid if you don’t completely understand documentation comments yet. They will be discussed further in lessons.

Note that in this course we use comments in the theory and practice lessons to explain how and why our code works.

Also, remember, a large number of comments in a program does not mean that it will become clear. The code may be changed, and the comments become outdated. To reduce the number of comments, you must write self-documenting code, which is not simple for novices. Throughout this course, we will discuss how to achieve it.
