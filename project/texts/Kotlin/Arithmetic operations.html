In real life, we often perform arithmetic operations. They help us to calculate the change from a purchase, determine the area of a room, count the number of people in a line, and so on. The same operations are used in programs.

Binary operators

Kotlin provides five operators to perform arithmetic operations:

  • addition +
  • subtraction -
  • multiplication *
  • integer division /
  • remainder % (modulus)

The operators are called binary because they take two values as operands.

The following example prints the results of addition, subtraction, and multiplication.

println(13 + 25) // prints 38
println(20 + 70) // prints 90

println(70 - 30) // prints 40
println(30 - 70) // prints -40

println(21 * 3)  // prints 63
println(20 * 10) // prints 200

The / operator divides the integer part of two numbers; the fractional part is discarded.

println(8 / 3)  // prints 2
println(41 / 5) // prints 8

The % operator finds the remainder of a division.

println(10 % 3) // prints 1, because 10 divided by 3 leaves a remainder of 1
println(12 % 4) // prints 0, because 12 divided by 4 leaves no remainder

Writing complex expressions

Arithmetic operations can be combined to write more complex expressions:

println(1 + 3 * 4 - 2) // prints 11

The calculation order coincides with the rules of arithmetic operations. Multiplication has a higher priority level than addition and subtraction, so the operation 3 * 4 is calculated first.

Note: At this moment you can consider an expression as any part of a code that produces a value (e.g. 14 + 5 or 15 - 3 * 4). In the following lessons, we will learn about expressions in more details.

To specify the order of execution, we can use parentheses, as in the following:

println((1 + 3) * (4 - 2)) // prints 8

As in arithmetic, parentheses can be nested. You can also use them for clarity.

For example, let's consider a program that prints all the digits of the number 54 in reverse order (the first is 4, and the last is 5). We will use arithmetic operations to extract its digits.

fun main(args: Array<String>) {
    println(54 % 10) // it prints 4
    println((54 / 10) % 10) // it prints 5
}

The program outputs:

4
5

Unary operators

A unary operator takes a single value as the operand.

  • The unary plus indicates a positive value. It is an optional operator, so try not to use it in practice.
println(+5) // prints 5

  • The unary minus negates a value or expression.
println(-8)  // prints -8
println(-(100 + 4)) // prints -104

They both have a higher level of precedence than the multiplication and division operators.

Order of precedence

Here is a list of the order of precedence of all the above considered arithmetic operators, including parentheses. The list is sorted from the highest to the lowest priority.

  1. parentheses
  2. unary plus/minus
  3. multiplication, division, and modulus
  4. addition and subtraction

Please memorize this list.
