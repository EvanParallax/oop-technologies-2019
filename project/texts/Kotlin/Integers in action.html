Basic information about Int and Long types

Integer numbers are often used in real programs. As you may recall, Kotlin has several types to store them. The most used types are Int (32 bits) and Long (64 bits). You can perform all types of arithmetic operations (+, -, *, /, %) with variables of these types. Let's look at some examples.

val two = 2  // Int
val ten = 10 // Int

val twelve = two + ten // 12
val eight = ten - two  // 8
val twenty = two * ten // 20
val five = ten / two   // 5
val zero = ten % two   // 0, no remainder

You probably already know what these operations actually do. The same operations work with the Long type. To declare a variable of the Long type, you may choose one of the following ways:

val longNumber1 = 1_000_000_000_000_000
val longNumber2: Long = 1_000_000
val longNumber3 = 1000L

val result = longNumber1 + longNumber2 - longNumber3
println(longResult) // 1_000_000_000_999_000

The value longNumber1 is Long because it is greater than the possible value of the Int type. The value longNumber2 is Long because we specified the type. The value longNumber3 is Long because we tagged the value with the suffix L (Long). The result is Long because it is the result of the sum of three Longs.

Use Longs only if it is really necessary (to process large values).

Reading numbers from the standard input

As a rule, to solve a problem you need to read some data from the outside world, process it, and output the result. For example, the following program reads two numbers from the standard input, adds them, and outputs the sum.

import java.util.Scanner

fun main(args: Array<String>) {
    val scanner = Scanner(System.`in`)

    val a = scanner.nextInt()
    val b = scanner.nextInt()

    val sum = a + b

    println(sum)
}

This simple code uses Scanner to read data. It works for positive, negative and zero numbers because the Int type supports all of them.

If we know that the input numbers can be quite large, we can read Long's instead of Int's:

val a = scanner.nextLong()
val b = scanner.nextLong()

No more lines need to be changed in this code.
