Conventions

Perhaps you know that programmers spent a lot of time reading existing code. If a program has unclear code, it is difficult to understand and develop it in the future. So, just a working program might not be enough.

If you have a team consisting of several programmers, it is very important to write readable and standardized code. Ideally, all your code should look as if only one programmer wrote it. Then, it will be easy for other team members, especially new ones, to read and understand it.

Code style conventions partially help us to standardize and support well-readable code. These conventions are more like recommendations than strict rules. If a programmer follows them, his code will be more clean and consistent.

Kotlin has its own conventions, which can be found here. We will follow them in all our examples and exercises. We recommend you do the same. Do not try to learn all the conventions at once; just open them from time to time after learning some new concepts. We will also give parts of these conventions in this course.

Follow your first conventions

Let's learn a few code conventions for Kotlin.

  1. It is recommended to use 4 spaces for indentation instead of tabs.
  2. In many cases, semicolons (;) are optional, so omit them wherever possible.
  3. Put the opening curly brace at the end of the line where the block begins, and the closing brace at the start of a separate line.

As an example, the program bellow follows both conventions

fun main(args: Array<String>) { // opening curly brace
    println("Hello, World!") // this statement is offset by 4 spaces and has no ; at the end
} // closing brace

Congratulations, you are already following the conventions! Continue in the same way to always write wonderful code. And remember: good code is like a good joke: it needs no explanation.
