The logical type

Boolean is a data type that has only two possible values: true and false. This is also known as the logical type.

val t = true  // t is true
val f = false // f is false

println(t) // true
println(f) // false
Remember, you cannot assign an integer value to a Boolean variable. In Kotlin, 0 is not the same as false.

This type is often used in programming (not only in Kotlin) to represent two opposite states such as:
  • a store is open (open - true, closed - false)
  • a plane is flying (yes - true, no - false)
  • travel is allowed (yes - true, no - false)
  • an option is enabled (yes - true, no - false)
  • and so on.

Logical operators

Variables of the Boolean type are often used to build logical expressions using logical operators. Kotlin has four logical operators NOT, AND, OR and XOR:

  • NOT is a unary operator that reverses the Boolean value. It is denoted as !.
val f = false // f is false
val t = !f    // t is true
  • AND is a binary operator that returns true if both operands are true, otherwise, it is false. It is denoted as &&.
val b1 = false && false // false
val b2 = false && true // false
val b3 = true && false // false
val b4 = true && true  // true
  • OR is a binary operator that returns true if at least one operand is true, otherwise, it returns false. It is denoted as ||.
val b1 = false || false // false
val b2 = false || true  // true
val b3 = true || false  // true
val b4 = true || true   // true

  • ﻿XOR (exclusive OR) is a binary operator that returns true if the Boolean operands have different values;  otherwise, it is false.
val b1 = false ^ false // false
val b2 = false ^ true  // true
val b3 = true ^ false  // true
val b4 = true ^ true   // false

The XOR operator is used less often than the others. Just remember that Kotlin has it. If you really need it, you can use it

Precedence of logical operators

Below are the logical operations sorted in order of decreasing priority in expressions:

  • ! - NOT
  • ^ - XOR
  • && - AND
  • || - OR

So, the following variable is true:

val bool = true && !false // true, because !false is evaluated first

To change the order of execution you can use parentheses (...).

Example: Hiking

As an example, let's write a Boolean expression that determines the possibility of trekking in summer and in other seasons.

val cold = false
val dry = false
val summer = false // now is autumn, suppose

val hiking = (dry && !cold) || summer // true, let's go hiking!

Do not get confused by the expression above; otherwise, you will go on a hike in bad weather :) A programmer should understand not only arithmetic, but also logical operations.
