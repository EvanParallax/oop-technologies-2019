In this lesson, we will look at what functions are and how they should be used. A function is a sequence of instructions whose execution is invoked from one or more locations in a program by its name. Functions represent subprograms that do something self-contained and useful, e.g. printing data to the standard output, calculating a square root and so on.

The picture below shows how you should understand functions by the end of this lesson. It is just a "box" that processes input arguments (input data) and produces either a useful result or nothing.

Arguments of functions

When we want to use a function, we invoke (or call) it using its name followed by parentheses. If a function takes one or more arguments (input data), they should be passed within the parentheses.

For example, below we invoke the println function with a single string argument.

val text = "Hello"
println(text)

This function can also not take arguments at all to print a new line.

println()

So, in its general form, a function can be invoked like this:

function1() // invoke function1 without an argument
function2(arg1) // invoke function2 while passing an argument
function3(arg1, arg2) // invoke function3 while passing two arguments
// ... and so on

where function is the name of a function.

Producing a result

Some functions may not only take an argument but also produce (return) a single result. This result can be assigned to a variable.

val result = function(arg)

Functions that take arguments and produce a result look like regular math functions.

For example, let's consider a math function that returns the absolute value of a number:

val number = -10
val nonNegNumber = Math.abs(number) // it takes -10 and returns 10

One benefit of using functions is that you don't need to implement anything; just invoke a function and it works :)

Note: The name of the abs function is written after the dot symbol. The reason is that Math groups multiple functions together, and to invoke one of them we should write the name of the group. We won't go over it in detail right now; just keep in mind that you will see similar things in our examples and practice problems.

Looking ahead, this is a good time to mention that in Kotlin, all functions actually return a result, even the println function.

val result = println("text")
println(result) // kotlin.Unit

This result is a special value called Unit, which means "no result".
