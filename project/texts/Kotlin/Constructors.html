We know how to declare simple classes with simple properties. Now let's talk about one more kind of class members – a constructor.

Default constructor

Constructors are class members that initialize a new object of the class. In other words, constructors set a new object state by setting its properties. So a constructor is invoked when you create an object.

Remember how we create objects? We write the class name and empty parentheses after it. It's exactly the constructor call and it's like calling the function with no arguments. Actually, it's because when there is no explicitly defined constructor, there is an implicit default constructor that just creates an object and doesn't have any logic inside.

Custom constructor

Often you know the object properties before you create it. To reduce and beautify your code, the properties can be set in the constructor. Just make the constructor receive the parameters.

As an example, let's use the Size class:

class Size {
    var width: Int = 0
    var height: Int = 0
}

Add the following constructor:

class Size {
    var width: Int = 0
    var height: Int = 0

    constructor(_width: Int, _height: Int) {
        width = _width
        height = _height
    }
}

And now you are able to create properly set objects in one line:

val size1 = Size(3, 4)
val size2 = Size(5, 1)

So to declare a custom constructor, write the constructor keyword inside the class and parathesis with its arguments after the keyword. Then write curly braces with the constructor logic inside them. Sometimes such constructors are called secondary ones.

Note that you either use an implicit constructor or declare your own, you can't use the both at the same time. You can create a replica of the default constructor explicitly, though (empty curly braces can be removed):

class Size {
    var width: Int = 0
    var height: Int = 0

    constructor() {
    }
}

The this keyword

Inside the class code, you can also access the object members using the "dot and the member name" syntax. Since usually you don't know the object name, there is a special keyword considered as the current object – this. For example, it can be used to name the constructor arguments as the class properties. Let's change the Size class:

class Size {
    var width: Int = 0
    var height: Int = 0

    constructor(width: Int, height: Int) {
        this.width = width
        this.height = height
    }
}

Omitting default values

Classes in Koltin have useful feature: you don't have to declare a default property value if the value is assigned in the constructor of the class:

class Size {
    var width: Int
    var height: Int

    constructor(width: Int, height: Int) {
        this.width = width
        this.height = height
    }
}

In this case, we can change vars to vals and a Size object will be immutable. Let's add another property just for illustration:

class Size {
    val width: Int
    val height: Int
    val area: Int

    constructor(width: Int, height: Int) {
        this.width = width
        this.height = height
        this.area = width * height
    }
}

Primary constructor

Another syntactic sugar Kotlin provides is a primary constructor. It allows to move the constructor keyword and the constructor arguments right after the class name:

class Size constructor(width: Int, height: Int) {
    val width: Int = width
    val height: Int = height
    val area: Int = width * height
}

The keyword of the primary constructor can be removed:

class Size(width: Int, height: Int) {
    val width: Int = width
    val height: Int = height
    val area: Int = width * height
}

Moreover, you can put simple properties declarations inside the primary constructor. Firstly, let's move one property:

class Size(val width: Int, height: Int) {
    val height: Int = height
    val area: Int = width * height
}

And finally - the remaining one:

class Size(val width: Int, val height: Int) {
    val area: Int = width * height
}

Single line classes

If there are no other class members remaining (imagine the area property is missing), as always, we can omit empty curly braces:

class Size(val width: Int, val height: Int)

You can see such classes frequently in real life. For example, classes that just store data are defined like this.

So a primary constructor is a handy way to concisely define a class.

Conclusion

  • Any Kotlin class has a constructor to initialize its objects.
  • A constructor is available for invocation under the class name.
  • There are different ways to declare a constructor but a primary constructor is the most concise.
  • If a class has no custom constructors, then there is an implicit default constructor with no arguments and it does nothing.
