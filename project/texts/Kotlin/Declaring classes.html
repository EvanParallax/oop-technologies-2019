When programmers implement an object-oriented program, they often use already defined classes to build the program logic. For example, an object of the Scanner class allows you to get the next word in the console input, and this word is an object of the String class. But also programmers frequently need to declare some program-specific classes. In this topic, we will discuss ways how to create your own simple class and how to use it.

Declaring new classes

In order to declare a new class, you should write the class keyword and the class name following it. To start, let's declare a class named Emptiness:

class Emptiness {
    // empty body
}

In Kotlin, when a class has an empty body, curly braces can be omitted. So the same class definition is the following:

class Emptiness

Classes can be declared in .kt files. Usually, they are declared at the top level of a file but nothing forbids you to declare them in other places, e.g. inside the function.

Object creation

What's the purpose of declaring a class? Briefly, each defined class introduces a new type. So you can create objects of a class and store them as variables and values. Sometimes standard types are insufficient so that opportunity is handy. There will be a few examples to make it clearer.

To create an instance of a class, simply write the class name and empty parentheses. In other words, the syntax is the same as the one for calling a function. As an example, let's do that for the Emptiness class:

val empty: Emptiness = Emptiness()

We have just assigned a new object of the Emptiness class to the empty variable. The variable has the Emptiness type so you can't reassign an object of any other type to it. As a general rule in Kotlin the explicit type directive can be omitted:

val empty = Emptiness()

Class member

A class body can include class members. For now, we will talk only about properties that replace fields in Kotlin: they can store data too. Other advantages of properties and other class members will be discussed in the next topics.

Note that all class members are optional. For example, the Emptiness class has no properties.

Writing properties

Properties are pretty similar to variables and values. If you want to assign values to a property during runtime, you declare it as a var, otherwise, a val is your choice.

Also, a property has a strict type. The type can be any. It can be a standard type like a number or a string or a custom type. So a property type can be your own class and even the same class where the property is declared.

A property that just stores data must somehow receive an initial value. For example, the property can be set in the class body.

A class can have as many properties as you need.

Now let's declare a class for representing a patient in a hospital information system:

class Patient {
    var name: String = "Unknown"
    var age: Int = 0
    var height: Double = 0.0
}

Here we see that the Patient class has three properties that can be reassigned: name, age, and height.

Each object of the class has the same set of fields, but the values of the fields can differ from object to object.

Accessing properties

Now we will discuss how to get access to object properties. First of all, create an object:

var patient = Patient()

We haven't changed the object properties yet, so they now have initial values. To get those properties' values, write a dot and the property name after the object name:

println(patient.name) // prints "Unknown"
println(patient.age)  // prints "0"

Changing properties

Consider the following program. The program creates two patients and sets their properties, then it prints the properties.

class Patient {
    var name: String = "Unknown"
    var age: Int = 0
    var height: Double = 0.0
}

fun main() {
    val john = Patient()
    john.name = "John"
    john.age = 30
    john.height = 180.0

    val alice = Patient()
    alice.name = "Alice"
    alice.age = 22
    alice.height = 165.0

    println("${john.name}: ${john.age} yrs, ${john.height} cm")
    println("${alice.name}: ${alice.age} yrs, ${alice.height} cm")
}

The output is:

John: 30 yrs, 180.0 cm
Alice: 22 yrs, 165.0 cm

Note that when we assign properties of Alice, properties of John remain unchanged.

Also, you should understand that the val before alice means that we can't reassign a Patient to alice. But we can reassign properties of alice because they are marked as vars inside the Patient class.
